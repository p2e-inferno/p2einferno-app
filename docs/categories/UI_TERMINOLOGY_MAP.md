# P2E Inferno UI Terminology Map

> Last Updated: March 11, 2026
> Category: UI Element and Terminology Map
> Purpose: A KB-safe translation layer between user wording, product terminology, and common UI labels in P2E Inferno. This document helps AI systems answer questions when the user, the interface, and the internal product language use different terms for the same thing.

## How To Use This Document

Use this document to answer questions like:

- Is pullout the same as withdrawal?
- What does DG Nation mean in the app?
- Is the vendor the same as token trading?
- What does Verify Identity refer to?
- Is a cohort different from a bootcamp?

This document is especially useful when:

- a user uses informal wording
- a support answer needs to map one label to another
- the app uses a branded term instead of a generic one

Do not use this document as the main source for:

- click-by-click instructions
- deep troubleshooting
- security-sensitive access logic

Those belong in other KB sources.

## Core Translation Principle

Users often describe features by:

- what they want to do
- what they saw on screen
- what they think the feature is called

The app may describe the same thing differently.

This map helps normalize those differences so support and AI guidance can answer accurately without forcing the user to know the exact in-app wording first.

## Rewards, Value, and Membership Terms

### DG

Preferred platform term:

- DG
- DG Token

Common user equivalents:

- reward token
- app token
- DG points

Meaning:

- the platform's ecosystem and reward token concept

Common confusion:

- users may mix DG with xDG, but they are not the same thing

### xDG

Preferred platform term:

- xDG

Common user equivalents:

- points
- experience points
- in-app balance

Meaning:

- the platform's branded in-app progression value tied to participation and completion

Common confusion:

- users may call all earned value "DG" even when the app is referring to xDG

Internal equivalent:

- XP

Response rule:

- understand XP and xDG as the same underlying in-app progression value
- prefer xDG in user-facing answers unless explicitly clarifying the synonym

## Answering Guidance

When a user uses a vague or mismatched term:

1. identify the likely platform term
2. answer using both the user's wording and the platform wording
3. clarify gently if the two are not exactly the same

Good examples:

- "If by withdrawal you mean DG pullout, that lives in your profile area."
- "If by token market you mean the DG Token Vendor, use the vendor page in the lobby."
- "If by session you mean your cohort, that is the scheduled run of the bootcamp you joined."
- "If by proof you mean a task submission, the app may still show it as pending verification until review is complete."

## Scope Guardrails

This terminology map intentionally avoids:

- internal component names
- admin-only terminology
- sensitive security logic
- implementation-specific API references

It is meant to improve retrieval quality by helping the KB translate safely between user language and platform language.
