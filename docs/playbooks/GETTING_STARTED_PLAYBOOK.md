# P2E Inferno Getting Started Playbook

> Last Updated: March 10, 2026
> Purpose: A practical starting guide for new users entering the P2E Inferno ecosystem. This playbook explains how to get oriented, what to do first, how the major parts of the app connect, and how to build early momentum without getting confused.

## Who This Is For

This playbook is for:

- new users joining P2E Inferno for the first time
- users who understand the app at a high level but do not know what to do next
- support or AI assistants guiding a user through their first meaningful actions

It is designed to answer a simple question:

**How do I get from “I just signed in” to “I am actively progressing inside the app”?**

## The Big Picture

P2E Inferno is not just one feature. It is an ecosystem made up of several connected layers:

- **Identity and access:** wallet connection, linked accounts, membership, verification
- **Learning:** bootcamps, cohorts, milestones, and milestone tasks
- **Action:** quests and daily quests
- **Progression:** rewards, streaks, achievements, and onchain proof
- **Economic engine:** the DG Token Vendor and its stage-based progression model

A new user does not need to master all of this at once.

The correct starting mindset is:

1. get your account and wallet setup right
2. understand the path you want to follow
3. complete a small number of real actions
4. build consistency
5. level into deeper features over time

## The First Goal

Your first goal in P2E Inferno is not to optimize rewards.

Your first goal is to become **operational** inside the ecosystem:

- you know which wallet the app is using
- you understand where to find your next action
- you can complete at least one meaningful task or proof-of-work step
- you understand how progression works well enough to avoid getting stuck

Once that foundation is in place, everything else becomes much easier.

## What To Set Up First

## 1. Connect to the app first, then set up wallet usage correctly

Before talking about self-custody wallet setup, the first real step is to connect to the app.

Users can enter P2E Inferno through:

- email
- an existing Ethereum wallet
- supported social account login such as Farcaster

This matters because P2E Inferno uses Privy. That means a user can begin using the app even if they do not yet have an external wallet ready, because Privy can provide an embedded wallet experience.

So the correct starting mental model is:

1. connect to the app
2. understand which wallet the app is using
3. decide whether to keep using the embedded wallet temporarily or link an external wallet
4. complete profile and account linking
5. move into the first onboarding quests

A user does not need to arrive with a self-custody wallet already set up in order to begin.

## 2. Understand the difference between embedded wallet access and external wallet readiness

New users can start with the embedded wallet, but they should understand its role clearly.

The embedded wallet is useful because:

- it lets the user get started quickly
- it reduces first-day friction
- it works as a fallback when no external wallet is available on the device

However, an external self-custody wallet is still strongly recommended for most serious quest participation.

That is especially important when:

- a quest requires the user to interact with onchain apps outside P2E Inferno
- the user will move across multiple Web3 apps or protocols
- the user wants to avoid confusion between embedded-wallet actions and external-wallet actions

Best practice for most users:

- connect to the app first
- then link or set up an external wallet as early as possible
- use that external wallet consistently for broader onchain activity

This reduces wallet mix-up and makes off-platform quest tasks much easier.

## 3. On mobile, use the wallet's in-app browser for the cleanest Web3 experience

This is one of the most important onboarding rules for mobile users.

On mobile, users should be guided to open P2E Inferno from the in-app browser inside their wallet app whenever they want to use an external wallet for Web3 interactions.

Examples include:

- MetaMask in-app browser
- Rabby in-app browser
- Coinbase Wallet in-app browser

Why this matters:

- normal mobile browsers such as Chrome or Firefox may not expose the external wallet correctly
- when the external wallet is not available in the browser context, the app may fall back to the embedded wallet
- that can create confusion about which wallet is active for signing, access, or quest completion

Best practice for mobile users:

- if they want to use their external wallet, open the app from the wallet app's browser
- do not assume a normal mobile browser will preserve the same wallet context

## 4. Complete account and profile linking early

After connecting, users should go to their profile and complete as much account linking as possible.

The profile is where users should review and connect important identity and access pieces such as:

- email
- wallet
- Telegram
- Farcaster, if they have one

This matters because early profile setup reduces friction later when quests or ecosystem actions depend on linked identity, communication, or wallet continuity.

The assistant should guide users to `/lobby/profile` early, not only when something is broken.

## 5. Complete any essential verification or access prerequisites

Some experiences may require:

- a connected wallet
- GoodDollar verification
- an active membership key
- a specific vendor stage

Not every user needs all of these immediately, but if a feature says you are ineligible, the reason is usually one of those.

## 6. Enter through the right doorway

Most new users should enter through one of these paths:

- **Bootcamp path:** for structured, guided learning
- **Quest path:** for action-based participation and challenge completion
- **Daily participation path:** for consistency, streaks, and lightweight ongoing progress

These are not separate universes. They reinforce each other.

## How the Main Parts of the App Fit Together

## Bootcamps

Bootcamps are the structured learning layer of the app.

They are best for users who want:

- guided progression
- a defined curriculum
- milestones and proof-of-work
- a cohort-based experience

Bootcamps are paid programs.

A user usually applies to a cohort first, then continues through a payment flow before enrollment is completed.

A bootcamp usually breaks learning into milestones and milestone tasks so users can move step by step instead of guessing what to do next.

## Quests

Quests are action-focused progression paths.

They are useful for:

- guided practical activities
- challenge-based participation
- moving from theory into execution
- earning rewards tied to actual completion

Quests are the free entry point for participation.

They do not use the same cohort payment flow as bootcamps.

Quests often complement bootcamps by turning learning into visible action.

## Daily Quests

Daily quests are the consistency layer.

They help users:

- show up regularly
- build habits
- maintain active engagement
- unlock lightweight but meaningful progress loops

Some daily quests may include eligibility requirements such as:

- vendor stage
- GoodDollar verification
- lock key ownership
- token balance requirements

This means daily quests are not just “mini tasks.” They can reflect where a user currently stands in the broader progression system.

## Daily Check-in

Daily check-in is the simplest recurring action in the app.

It is useful because it:

- reinforces consistency
- supports streak-building
- creates a low-friction daily habit
- helps users stay connected to the ecosystem even when they are not doing a larger task

For many users, check-in is the easiest first recurring behavior to establish.

## The DG Token Vendor

The DG Token Vendor is a foundational progression and participation system in the ecosystem, but it is not the only part of the app.

Its role is to provide:

- NFT-gated access
- structured participation
- stage-based progression
- points and fuel mechanics
- a transparent rule-based progression loop

The important thing for a new user is not to start with maximum complexity.

The important thing is to understand that the vendor is part of the app’s broader progression model. As users mature in the ecosystem, vendor-related mechanics become more important.

## What New Users Should Do First

For most users, this is the recommended order:

1. Connect to the app using email, wallet, or supported social login.
2. Identify whether the app is currently using the embedded wallet or a linked external wallet.
3. If possible, link or set up an external self-custody wallet early.
4. On mobile, use the wallet app's in-app browser when interacting with the app through an external wallet.
5. Visit `/lobby/profile` and complete important account linking such as email, wallet, Telegram, and Farcaster if available.
6. Complete the onboarding-style quests that establish account readiness, such as Rosy Beginnings and wallet setup.
7. Then move into broader quest, bootcamp, or daily participation flows.

## A Good First-Week Path

This is the healthiest first-week approach for a new user:

### Day 1: Get oriented

- connect to the app
- understand whether the current wallet is embedded or external
- if possible, link or prepare an external self-custody wallet
- if on mobile, open the app from the wallet's in-app browser for external-wallet flows
- visit `/lobby/profile`
- complete key account linking such as email, wallet, Telegram, and Farcaster if available
- explore the lobby
- identify whether you are here primarily for learning, quests, or broader ecosystem participation
- identify the onboarding quests that establish account readiness before moving deeper into the ecosystem

## The First Quests Should Build Account Readiness

Before moving into deeper quest participation, new users should establish a clean operating foundation.

The most important early quest pattern is:

- connect and link the needed accounts
- set up or link a usable self-custody wallet
- make sure profile-level identity is complete
- only then move into broader quest participation

Two especially important examples are:

- **Rosy Beginnings**
  - this is the account-linking and profile-readiness quest
  - it helps users complete the basic identity setup needed for clean participation
- **Set Up Wallet**
  - this is the self-custody wallet setup step
  - it matters because many broader Web3 quests depend on a wallet the user controls outside the app

These are not optional side details. They are foundational onboarding steps.

### Day 2: Start one guided path

- apply to a bootcamp, or
- start a quest path, or
- begin engaging with daily participation features

Do not start everything at once.

### Day 3: Complete one real action

Good examples:

- complete a simple milestone task
- complete a quest task
- complete a daily quest task
- finish a verification step that unlocks a blocked area

This is where the app starts becoming real.

### Day 4 to Day 7: Build consistency

During the rest of the first week:

- keep checking in
- keep completing the next visible step
- learn which requirements are gating your next action
- avoid jumping prematurely into advanced features you do not yet need

## How To Choose Your Main Path

## Choose bootcamps first if:

- you want structure
- you want guided learning
- you prefer weekly milestones and clearer educational progression
- you are newer to the subject matter

## Choose quests first if:

- you want challenge-based action
- you want to complete tasks more independently
- you want quicker loops of action and reward
- you are comfortable exploring through guided tasks rather than formal curriculum

## Choose the vendor path earlier if:

- you already understand why membership and gated participation matter
- you want to engage with the ecosystem’s deeper progression mechanics
- you are ready to think about stage progression, points, fuel, and vendor-linked participation

Even then, it should still sit inside a broader journey, not replace it.

## What Usually Confuses New Users

## 1. Wallet confusion

Users often assume:

- linked wallet = active wallet

That is not always true.

The app may use the embedded wallet when an external wallet is not available on the current device. That can affect feature access, signing, and membership behavior.

## 2. Membership confusion

Users often assume:

- if one wallet on the account holds a membership NFT, all actions will work the same way

That is also not always true.

Some checks are account-wide across linked wallets. Others require the active wallet itself to hold the key.

## 3. Reward-first thinking

Users sometimes jump straight to maximizing DG, xDG, or access without understanding the progression model.

That usually creates friction.

The better approach is:

- learn the system
- complete real actions
- understand the gates
- then optimize later

## 4. Treating every feature as independent

Bootcamps, quests, daily quests, vendor progression, and membership are connected.

A user progresses best when they understand that these systems reinforce each other rather than compete with each other.

## What “Good Progress” Looks Like

A user is making good early progress when they can say:

- I know which wallet the app is using
- I know where my next task lives
- I have completed at least one real action
- I understand why a feature is blocked if I cannot access it
- I can distinguish between learning flow, daily engagement, and vendor progression

That is a much better marker of success than simply “I earned something once.”

## When To Go Deeper Into Vendor Progression

A user should spend more time on vendor-specific progression when:

- they already understand the app’s general flow
- they have established basic consistency
- they are ready to engage with membership-gated systems
- they want to understand stage progression, points, fuel, and vendor-linked eligibility

At that point, the vendor is no longer just a feature. It becomes part of the user’s longer-term progression strategy inside the ecosystem.

## Recommended Assistant Guidance

When guiding a new user, assistants should prioritize:

1. clarity over hype
2. next action over feature dumping
3. explanation of gates over generic encouragement
4. one clear path over too many options at once

The best support response for a new user usually answers:

- what should I do next
- why should I do that next
- what could block me
- how do I know I’m progressing correctly

## Short Version

If a user asks, “How do I start?” the best short answer is:

1. connect and understand your wallet setup
2. start with a guided path such as a bootcamp or quest
3. complete one real task
4. build consistency through daily participation
5. explore deeper systems like vendor progression once the basics are clear

That is the healthiest way to start in P2E Inferno.
